# 旅行准备与文化冲击（practice 精炼版）

> 目标：同一主题下激活两大子域（packing / culture shock），每天 5 条语块 + 1 骨架 + 1 迁移 + 1 升级。

## 1. 子主题聚焦
core: packing for a trip / cultural differences  
extension pool: must‑see countries / travel experiences（仅做迁移或例子来源）

## 2. 功能表达核心 10 选（本主题推荐先练）
| 功能 | 起手 | 本主题示例 |
|------|------|------------|
| principle | A simple rule is… | A simple rule for packing is to bring fewer but versatile items. |

| example | For example… | For example, I take two pairs of shoes: walk + nicer. |
| contrast | Unlike what I used to think… | … some SE Asian cities are very modern. |
| expectation | I expected…, but actually… | I expected Europe to be fully digital, but I often needed cash. |
| trade‑off | I’d rather sacrifice X… | … extra outfits to keep my luggage lightweight. |
| change | I’ve started to… | I’ve started to roll clothes instead of folding them. |
| transfer | I apply the same principle to… | … my daily schedule—drop single‑purpose tasks. |
| reflection | This opened my eyes because… | … assumptions don’t always match reality. |
| value | What really matters is… | … feeling flexible, not carrying “just in case” stuff. |

首轮先锁：avoid overpacking / roll clothes to save space / versatile items / more modern than expected / challenge my assumptions。

## 4. 双骨架示范
packing P-R-E-C-W:
Principle: A simple rule is to pack versatile items, not many.  
Reason: It cuts decision fatigue and prevents overpacking.  
Example: For five days I take two bottoms, four mix tops, walking + nicer shoes.  
Contrast: I used to add “just in case” outfits I never touched.  
Wrap: Now my luggage stays lightweight yet flexible.

culture shock P-R-E-(C)-Value:
Principle: Travel corrects assumptions.  
Reason: Media pictures differ from real use.  
Example: I expected Europe fully digital but often had to pay in cash.  
Contrast: Meanwhile Southeast Asian cities felt highly modern.  
Value: Test reality before repeating a belief.

## 5. 迁移示例
Packing → 时间管理：Strip out single‑purpose items → Strip out single‑purpose calendar blocks.  
Culture → 学习：Test assumptions with reality → Try a mini‑assignment before committing a full course.

### 6.a 旧语块复习与轮换规则（“2 旧 3 新” 具体执行）
> 目标：保持“正在巩固”的活动池小（≈12–15 条），避免旧语块全部挤占名额，也防止新语块过量导致遗忘。

#### 分层标记体系（直接在语块列表后加符号）
- (0) 新抽取未用过（new）
- (*) 已在输出/录音中自然使用 1 次（used once）
- (**) 使用 ≥2 次但还未迁移 / 还偶尔卡顿（working）
- (t) 已用过至少 1 次迁移（transfer hit）
- (✔) 毕业：满足“3 次不同语境 + 1 次迁移”

在 `复用语块` 原始 15 条后面加符号即可。例如：
```
avoid overpacking (**) (t)
roll clothes to save space (*)
versatile items (✔)
challenge my assumptions (*)
more modern than expected (**) (t)
```

#### 每日选 5 条算法（伪代码）
```
active_pool = 所有(0|*|**) 语块（不含 ✔）
1. 若 active_pool < 8 → 从原始/备用列表补 1–2 条 (0)
2. old1 = 最近 2 天未选 & 标记 (**) 优先
3. old2 = (t) 标记少的 (*) 或 (**) （保持迁移多样）
4. new1/new2/new3 = 先选 (0)，不足再选 (*) 中仍有犹豫的
5. 如果某条连续 2 天仍卡壳 → 第二天强制 old slot1 再复现
```

#### 毕业条件（标 ✔ 并移出 active）
1. 无提示 1 秒内说出
2. 两个不同骨架里出现过（例如 一次在 P，一次在 Example）
3. 至少 1 次迁移句中用到（加 (t)）
→ 标记 ✔，后续只在周复盘时整体回顾，不再占每日 5 个名额。

| 3 | 若 (0) >5 → 只保留 3 个最有迁移潜力 | 控制输入节奏 |
| 4 | 统计 ✔ 总数（写入 progress_log notes） | 可见进展 |

### 6.b 10 分钟循环逐步详解 + 当日完整示例

#### 总览心智图（记住三条主线）
收集→输出→强化：
1 选材（挑 5）→ 2 释义热身 → 3 写骨架整合 → 4 口语录音 → 5 自评标错 → 6 升级句 → 7 迁移句 → 8 制少量卡。

---
#### 0–1 分钟：选 5 语块
- 目的：给接下来的输出“词汇钩子”，避免写骨架时空白。
- 操作：按 6.a 规则快速圈：2 旧 (**) + 1 已迁移不足 (*/**) + 2 新 (0)。
- 检查：不重复昨天全部 5 条；至少 1 条有迁移潜力（如可跨到时间管理/学习）。
- 常见失误：挑 5 条彼此关系弱 → 写骨架难以嵌入；全是新 → 记忆质量低。

#### 1–2 分钟：口头释义（不看中文）
- 目的：激活语义通道，确认是否真正理解而非视觉记忆。
- 操作：对每条用中文或更长英语 paraphrase；卡壳 >1s 标一个 “?”。
- 检查：5 条中 “?” ≤2；若 >2 → 减少当日新增量。
- 常见失误：直接重复原块不加解释 → 说明未内化。

#### 2–5 分钟：写骨架（mini P-R-E / P-R-E-C-W）
- 目的：把 5 条语块嵌进一个逻辑段落，建立“触发→输出”路径。
- 操作：先列 P / R / E 粗线，再把 3+ 语块塞入对应行；若有对比再加 C，最后 Wrap/Value 收束。
- 检查：
	- ≥3 语块已自然嵌入（非生硬堆砌列表）。
	- 每行 ≤1 主句（防止啰嗦）。
- 常见失误：为塞语块而造空壳句；忽略“原因”导致段落单薄。

#### 5–6 分钟：录音 30–40 秒
- 目的：训练流畅输出 + 回放自我监控。
- 操作：看骨架关键词，不逐字读；允许轻微重组。
- 检查：P-R-E 至少三段逻辑完整；停顿 >2s 不超过 1 次。
- 常见失误：逐字朗读写稿（缺少口语化）；语块顺序与骨架偏差过大（说明写稿未内化）。

#### 6–7 分钟：听并标 2 处（微诊断）
- 目的：精准微调而非泛泛“需要多练”。
- 操作：回放一次，只抓：① 用词不自然/直译；② 语法/时态错误。各标 1 处，用 * 或数字标记在骨架旁。
- 检查：只标 2 项（强制聚焦）。
- 常见失误：列一堆小拼写；忽略真正卡顿点（长停顿/结构混乱）。

#### 7–8 分钟：升级 1 句（旧→新）
- 目的：把“够用”句提升为“携带机制/对比/度量”的可迁移模式。
- 操作：挑刚才骨架中最普通的一句：加入 metric（数量 / 频率）、mechanism（导致结果的手段）、对比、条件或结果。
- 检查：升级后新增信息点 ≥1；删除了原句中 1 个模糊词（many / a lot / good 等）。
- 常见失误：只加 very / really；或仅换近义词不加结构信息。

#### 8–9 分钟：迁移 1 句
- 目的：锚定“跨场景检索”能力，防止语块被主题绑定。
- 操作：选 1 原则或结构（如 test assumptions…）→ 替换名词域到学习 / 时间 / 工作；保持逻辑因果。
- 检查：新句若给不熟悉主题的人读仍通顺；不出现原主题具体名词（除非作对比）。
- 常见失误：仅把 travel 换成 study，其余照搬 → 迁移失败。

#### 9–10 分钟：制 2–3 张卡（最小必要）
- 目的：为易滑落或升级信息做“延时提醒”，不是记录全部。
- 操作：
	- 1 张 Cloze：挖升级句里的 metric/mechanism。 
	- 1 张 Transfer Prompt：提示原则 + 新领域应用简述。
	- （可选）1 张 Cloze 对比句（expectation vs reality）。
- 检查：不做原样抄录卡；无同时挖 5 个空的噪声卡。
- 常见失误：为“安心”把 5 条语块全部做卡，导致复习过载。

### 6.c 常见疑问解答：释义 / 录音 / 升级句

#### (1) 1–2 分钟“口头释义”到底怎么做？中文还是英文？
目的：检测你是否真的“掌握概念”，而不是机械背词。允许使用：
1. 中文解释（更快暴露含糊点）
2. 简单英语重述（Paraphrase）
3. 场景微句（Mini-scenario）
顺序建议：中文/英语任选其一 + 场景一句。

五个层级（Ladder）——越往上越内化：
L1 直译（避免停留）：avoid overpacking = 避免打包太多东西。
L2 中文换词：刻意不塞“以防万一”衣服。
L3 英文释义：choose fewer items so the bag stays light.
L4 场景句：Before a 5‑day trip I set a 4‑top limit so I stop adding extras.
L5 原则化：A self‑imposed limit kills the “add one more” loop.

判断是否合格：
- 能用不同词重述（不能只重复 roll clothes to save space 原样）。
- 说的时候脑中浮现“画面/动作”而非单词拼写。

快速操作模板（任选其一）：
- 中文解释：这是 …（类别）+ 作用：… + 避免：…
- 英文：It means I … instead of … so …
- 场景：When I … I … so that …

示例 3 条：
roll clothes to save space → 我把衣服卷起来，压缩体积，留出 15% 空位。
more modern than expected → 实际比我原来印象里现代得多（说明“预期差”）。
leave buffer space → 故意不塞满，让行程中新加入的物品有容纳区。

#### (2) 录音 30–40 秒：读骨架还是扩展？
不是“朗读稿件”，而是“看提纲自由表达”。
操作：
1. 录前看一眼骨架各行关键词（P/R/E/C/W）。
2. 开录后只盯 1–2 个关键词；其余靠你当下组织。
允许扩展：
- 加 1 个数字（two pairs / four tops）
- 加 1 个微对比（I used to pack X but…）
- 加 1 个结果信号（so mornings are faster）
控制：不要临时讲到完全不同话题（偏离骨架）。
判断好坏：
- 听感像口语（有自然连接词 so / and / then），而不是平铺书面句。
- 含最少 3 条当天语块。
失败信号：
- 逐字照读（语调单一）
- 语块一个没用上（说明前 5 分钟白做）。

#### (3) 升级句：没有思路怎么办？
核心：选一句“模糊 + 常用 + 核心”的旧句，用 1 个“升级杠杆”加工。

八种常用升级杠杆（记 3–4 个先用）：
```
1 机制 Mechanism：by / which + explains how
2 度量 Metric：number / frequency / ratio / limit
3 对比 Contrast：instead of / rather than
4 条件 Condition：when / if / after
5 结果 Consequence：so / which means / thereby
6 范围 Scope：only / at most / no more than
7 具体化 Specificity：把 vague 词替换成 可数/可见 项（stuff → four mix-and-match tops）
8 约束 Trade-off：I sacrifice X to keep Y
```
升级算法：
Step1 找一条含模糊词（many / a lot / good / things / stuff / faster / better）。
Step2 选 1 个杠杆（机制/度量/对比…）。
Step3 补足信息 → 删除原句里 1 个模糊词。
Step4 读一遍：是否新增了“可验证”元素？是则成功。

示例（基句：It prevents overpacking.）
- 机制版：It prevents me from packing “just in case” outfits I never touch.
- 度量版：It cuts my list down to four mix-and-match tops.
- 对比版：Instead of stuffing backup outfits, I set a four-top ceiling.
- 条件+结果：When I set the limit first, I stop the late “add one more” spiral.
- 约束版：I sacrifice variety so my bag stays flexible and light.

再示例（文化差异基句：Cities were modern.）
- 具体化：Downtown cafés took mobile payments and next‑day delivery felt routine.
- 对比：Instead of dusty streets I imagined, I saw glass stations and QR screens everywhere.
- 机制+结果：Mobile payment ubiquity stripped away my “carry cash everywhere” habit.

模板速用（抄写后替换变量）：
```
By + 具体动作, I + 结果（mechanism）
I set + 数字/上限 + so / which keeps + 结果
Instead of + 旧行为, I + 新策略 + so …
When I + 条件, I avoid + 问题
By removing + A, I gain + B without + C
```
升级自检清单：
- 有一个“可验证”的新增元素（数字 / 行为 / 具体物品 / 机制）
- 原句至少去掉 1 个模糊词
- 朗读时信息密度 ↑ 但仍一口气读完

练习（自己先做 → 看参考答案）：
```
1 旧：I bring many shirts.  
	参考升级：I cap myself at four mix-and-match tops so mornings take no thought.  
2 旧：Cities were modern.  
	参考升级：Every café took mobile pay, which killed my “carry cash” assumption.  
3 旧：I learned a lot.  
	参考升级：I replaced three vague “maybe” items with one rule: only pack what solves two situations.
```
首周策略：每天只升 1 句；7 天后再回看最常用旧句，挑 1–2 个重复升级到不同版本（metric / contrast）再选最佳入库。

> 记忆口令：升级 = 模糊句 + 杠杆（机制/度量/对比/条件…） → 去模糊 + 加可验证信息。
